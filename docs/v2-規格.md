## 1. 產品目標（一句話）

宜大校本部周邊（**預設半徑 500 m**），依**評價 + 距離 + 預算 + 分類**找店；**地圖**顯示全部符合條件店（未營業標「休息中」）；**轉盤**只在**營業中**且符合條件的店裡抽選（含「其他」分類），候選 **Top 40**。

## 4. 預算／平均消費（約 100 元）

### 4.1 Google 能給什麼

Places API (New) **沒有「人均 100 元」**，只有 `priceLevel` 四檔：

| Google enum | `price_level` | v1 顯示（`format_avg_spend`） |
|-------------|---------------|-------------------------------|
| INEXPENSIVE | 1 | $100–300 |
| MODERATE | 2 | $300–600 |
| EXPENSIVE | 3 | $600–1,200 |
| VERY_EXPENSIVE | 4 | $1,200+ |

v2 已透過 `providers.py` 抓取 `priceLevel` → JSON `price_level`。

### 4.2 限制

1. 無法精準「100 元」— 只能篩 **平價檔（level 1）**
2. 很多店 `price_level: null`（公開快取中覆蓋率偏低）
3. 區間依 Google 全球標準，非宜大學生午餐價

### 4.3 v2 實作方案

**P0（2026-06-25 已接線）**

- 新增篩選：`max_price_level=1` + `max_avg_spend_ntd`（OR；overlay `avg_spend_ntd` 優先顯示「約 $N」）
- CLI：`--max-avg-spend-ntd`；Streamlit：人均上限 number_input
- 選項：**「含未標價位」**（預設開）

## 5. 營業時間連動

### 5.1 Google 能給什麼

| 欄位 | 用途 |
|------|------|
| `currentOpeningHours.openNow` | 查詢當下是否營業 |
| `regularOpeningHours` | 每週固定時段（可離線重算） |
| `businessStatus` | `OPERATIONAL` / `CLOSED_TEMPORARILY` / `CLOSED_PERMANENTLY` |

Field mask 範例（擴充現有）：

```
places.regularOpeningHours,places.currentOpeningHours,places.businessStatus
```

> **計費：** 營業時間欄位屬 Places API **Enterprise SKU**，比現行欄位貴。建議 **fetch 時寫入快取，查詢時本地計算**，不要每次使用者操作都打 API。

### 5.2 實作原則

```
fetch 時：存 regularOpeningHours + businessStatus 進 JSON
查詢時：opening_hours.py 用 Asia/Taipei 從 regular 算出 is_open_now
轉盤：只餵 is_open_now == True 的候選
```

**禁止：** 只在快取存 `openNow` 布林值（隔天即失效）。

### 5.3 邊界情況

| 狀況 | 建議處理 |
|------|----------|
| 無營業時間資料 | `is_open_now = null` → 轉盤**保留**（避免候選過少） |
| `CLOSED_PERMANENTLY` | 地圖與轉盤皆排除 |
| `CLOSED_TEMPORARILY` | 地圖顯示並標「休息中」；轉盤排除 |
| 國定假日／特殊日 | v2.1 | `specialDays` + `tw_holidays.py`；無覆寫時沿用平日時段 |

### 5.4 UI 行為（已定案）

- **轉盤：** 未營業一律不進候選
- **地圖列表：** **顯示**未營業店，狀態欄標 **「休息中」**（`is_open_now == False`）；營業中顯示「營業中」；無資料顯示「未知」
- **永久歇業：** 地圖與轉盤皆不顯示

### 5.5 程式改動點

| 檔案 | 變更 |
|------|------|
| `models.py` | `regular_opening_hours`, `business_status`, `is_open_now`（runtime） |
| `providers.py` | 擴 field mask、解析 OpeningHours |
| `opening_hours.py`（新） | `is_open_now(hours, now, tz)` |
| `service.py` | `rank_nearby(..., open_only: bool)` — 僅轉盤路徑設 `open_only=True` |
| `streamlit_app.py` | 地圖表加「營業狀態」欄；`_render_wheel_selector` 先 filter 營業中再 `head(30)` |
| `fetch_places_to_json.py` | 序列化營業時間欄位 |

## 6. 資料池：1 km 內盡量抓滿

### 6.1 目標

- 中心：宜大校本部 `24.7464, 121.7457`
- 半徑：**1000 m**
- 策略：**盡量抓滿** Google 可回傳的 unique 餐飲 POI（去重後 haversine ≤ 1 km）
- **驗收**：fetch 後報告實際 N 家與覆蓋率；**不要求固定 300 家**

### 6.2 實測

| 輪次 | 策略 | ≤1 km 家數 | price_level | 營業時間 |
|------|------|-----------|-------------|----------|
| 2026-06（初） | 格網 5×5 Nearby | **123** | 81/123 | 122/123 |
| 2026-06（加強） | 格網 6 + 逐類型 + Text Search | **300** | 150/300 | 282/300 |

加強版已達腳本 `--target` 上限（300）；**驗收仍為盡量抓滿 + 報告 N**，非固定家數。

### 6.3 抓取策略

```powershell
cd 校園美食地圖_v2
$env:PYTHONPATH = "src"
$env:GOOGLE_MAPS_API_KEY = "你的金鑰"

python scripts/fetch_places_to_json.py `
  --lat 24.7464 --lon 121.7457 `
  --radius-m 1000 `
  --grid 6 `
  --max-pages 3 `
  --out data/places_cache.json

python scripts/enrich_food_groups.py data/places_cache.json
```

| 階段 | 說明 |
|------|------|
| 格網 Nearby | 6×6 子圓心 |
| 逐類型 Nearby | 36 種 `includedTypes` 各搜一輪 |
| Text Search | 宜大／神農路關鍵字補漏 |
| 後處理 | haversine **硬篩 ≤ 1.0 km** |

`--target` 僅為腳本**提前停止上限**（預設 300），非產品 KPI。

### 6.4 若仍想多撐一點

1. `--grid 7` 或 `--max-pages 5`（多燒 API）
2. `--no-text-search` 關閉可省額度（通常較少）
3. 接受 Google 索引飽和即停止

### 6.5 轉盤與資料池的關係

- **資料池** = fetch 後 1 km 內全部店家（加強版 **300**；初版 123）
- **轉盤** = 通過「營業中 + 分類 + 預算 + 半徑 + 評論門檻」後，取綜合分 **Top 40**

## 7. 分類（十五類 + 其他）

### 7.1 使用者分類

側欄 `multiselect` 與 CLI `--food-group` 使用下列名稱（**必須完全一致**）：

| 分類 | 說明 |
|------|------|
| 滷味類 | 滷味攤／店 |
| 下午茶咖啡廳類 | 咖啡、茶飲、甜點、早午餐 |
| 火鍋類 | 火鍋、涮涮鍋、石頭鍋 |
| 牛排館類 | 牛排館 |
| 麵食類 | 牛肉麵、拉麵、米粉等 |
| 日式料理類 | 壽司、居酒屋、日式定食 |
| 韓式類 | 韓式料理 |
| 素食蔬食類 | 素食、蔬食 |
| 點心包子類 | 包子、水餃、點心 |
| 義式披薩類 | 義式、披薩、義大利麵 |
| 百匯自助餐類 | 百匯、自助餐 |
| 熱炒合菜類 | 熱炒、合菜、串燒 |
| 燒腊港式類 | 燒腊、港式、粵菜 |
| 便當類 | 便當、盒飯、定食、粥品 |
| 炸物類 | 炸物、雞排、鹽酥雞 |
| **其他** | 餐酒館、飯店、非美食 POI 等 |

快取家數統計見 `AI_HANDOFF.md` §3 或 `python scripts/stats_food_groups.py data/places_cache.json`。

### 7.2 為何不能直接用 Google `primaryType`

快取中實際類型範例：`hot_pot_restaurant`, `cafe`, `coffee_shop`, `fast_food_restaurant`, `snack_bar`, `taiwanese_restaurant`, `steak_house`…

Google **沒有**「滷味」「炸物」等專用 type → 需 **`categories.py` 對照表**（`primaryType` + 店名關鍵字）。

### 7.3 分類規則

- **自動分類：** `categories.py` → `classify_food_groups(category, name)`
- **人工覆寫：** `data/food_group_overrides.json`（key = place id；非美食強制「其他」、邊界店校正）
- **載入點：** `MockRestaurantProvider.list_restaurants()` → `data_overrides.apply_restaurant_overrides`
- 一店可屬多個 `food_group`（例：鹽酥雞 → 炸物類）
- 未命中 → **`其他`**

### 7.4 UI（已定案）

- Streamlit 側欄：`st.multiselect` 十五類 + **「其他」**（可多選）
- **轉盤：** 使用者有選分類時，候選 = 所選分類 **∪ 其他**（`wheel_food_groups()`）
- **地圖表：** 依側欄分類篩選；未選任何分類 = 顯示全部
- CLI：`--food-group 火鍋類 --food-group 牛排館類`（可重複）

### 7.5 資料模型

```json
{
  "id": "...",
  "name": "...",
  "category": "hot_pot_restaurant",
  "food_groups": ["火鍋類"]
}
```

`category` 保留 Google 原始 type；`food_groups` 為 v2 使用者分類（fetch 或 `enrich_food_groups.py` 計算）。

## 11. 風險

| 風險 | 影響 | 緩解 |
|------|------|------|
| Google 索引策略差異 | 初版 123 vs 加強版 300 | 以加強版快取為準；`--target` 為腳本上限 |
| `price_level` 大量 null | 預算篩選變形同虛設 | 預設「含未標價位」；長期補 `avg_spend_ntd` |
| 營業時間缺失 | 轉盤仍含未知店 | 保守保留；永久歇業一律排除 |
| API 費用上升 | Enterprise 欄位 | 僅 fetch 批次寫快取 |
| 分類誤判 | 滷味／炸物靠關鍵字 | 測試 + 手動校正表 |
